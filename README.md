# D-Link Switch Diagnostic Tool

**Автоматизация диагностики портов D-Link: 5–7 секунд вместо 5 минут ручной работы.**

Скрипт избавляет инженера от необходимости вручную подключаться к каждому свитчу по telnet/ssh при поиске проблем юзера. Достаточно ввести номер абонента — и скрипт сам найдёт оборудование, проверит порт, линк, ошибки, VLAN, ACL, загрузку CPU, а затем сходит на L3-шлюз и сверит ARP.

Реально используется в сети провайдера для ежедневной работы инженеров техподдержки.

---

## Что делает скрипт

При вводе номера юзера скрипт обращается к базе данных **MariaDB** и получает:

- Модель свитча
- IP-адрес юзера (внутренний/внешний/прямой внешний)
- Номер порта
- IP-адрес шлюза L3
- IP-адрес свитча
- Доп. IP-адрес юзера
- Тип DHCP (0/1)

Затем автоматически подключается по telnet к свитчу и запускает диагностику порта.
**Время выполнения — 5–7 секунд.**

---

## Что проверяет

| Параметр | Что показывает |
|----------|----------------|
| Link status | Есть ли линк на порту |
| MAC-адрес | Доходят ли настройки от маршрутизатора (сверка с таблицей MAC-адресов) |
| VLAN | Корректность номера VLAN и тип порта (trunk/access) |
| DHCP relay | Включён ли DHCP-агент на свитче |
| Errors on port | CRC-ошибки и другие проблемы на интерфейсе |
| Packets on port | Наличие трафика (RX/TX) |
| Cable diagnostic | Состояние медного кабеля юзера |
| CPU load | Текущая загрузка свитча |
| ACL | Есть ли разрешающие или запрещающие правила для порта |
| ARP (L3) | После диагностики скрипт идёт на шлюзовой свитч L3 и сверяет ARP-запись с MAC-адресом из L2-таблицы |

## Поддерживаемое оборудование

- D-Link DGS-1210
- D-Link 3028
- D-Link 3000-24TC
- D-Link 3120-24TC
- D-Link 3200-24

## Установка

```bash
# Клонировать репозиторий
git clone https://github.com/viMdi/MAIN_DIAGNOSTICL2
cd MAIN_DIAGNOSTICL2

# Создать виртуальное окружение
python -m venv myvenv

# Активировать
# Windows:
myvenv\Scripts\activate
# Linux/Mac:
source myvenv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Создать файл конфигурации из примера
cp cfg_example.py cfg.py
# Отредактировать cfg.py, указав реальные параметры подключения к БД и оборудованию

Использование
bash

python MAIN_DIAGA.py

После запуска ввести номер юзера. Скрипт покажет найденные записи и предложит запустить диагностику.
Результат — через 5–7 секунд.
Зависимости

    Python 3.6+

    MariaDB/MySQL

    Библиотеки: pymysql, pexpect (устанавливаются через requirements.txt)
